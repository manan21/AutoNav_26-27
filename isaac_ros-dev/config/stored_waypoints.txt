# stored_waypoints.txt — mission chain consumed by run_mission.sh
#
# Each non-blank, non-comment line is one leg of a sequenced mission.
# Legs fire top-to-bottom; the first leg that aborts stops the chain.
#
# Format:  <KIND> <a>, <b>     (two-arg legs)
#          <KIND>                (argless legs: GI, GR)
# Kinds:
#   ER  Euclidean Relative — desired nav_center travel (dx, dy) in meters,
#       relative to the robot's pose at the moment the leg starts.
#       Dispatched to send_goal.sh -r dx dy
#   EA  Euclidean Absolute — (x, y) in meters, in the robot's
#       initial map frame. Dispatched to send_goal.sh x y
#   GA  GPS Absolute — (latitude, longitude) in decimal degrees;
#       use negative values for S latitude / W longitude.
#       Dispatched to send_GPS_waypoint.sh lat lon
#   GI  GPS Insert — record the robot's current GPS position by averaging
#       10 /gps_fix samples and store it as the "return" waypoint. Place
#       at the top of a mission to capture the start pose.
#   GR  GPS Return — navigate back to the GI-recorded waypoint. Place at
#       the bottom of the mission for an out-and-back course.
#
# Lines beginning with '#' and blank lines are ignored.
# Numbers may be space- or comma-separated.
# ER 5.0, 0.0
